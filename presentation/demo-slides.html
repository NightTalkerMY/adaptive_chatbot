01 · Objective

What the task asked for

✓ Streaming REST API

One FastAPI POST endpoint returns incremental Server-Sent Events.

✓ Chat interface

A responsive Next.js client consumes and renders the live stream.

✓ Conversation memory

SQLite persists sessions and messages; history is replayed every turn.

✓ Quality assurance

Deterministic tests cover streaming, persistence, memory, and failures.

Bonus coverage: Docker Compose and a user-friendly adaptive interface.

03 · Streaming contract

One request, incremental events

metaSession ID, category, title, and suggested follow-ups
token *One event for each text chunk received from Gemini
doneSession ID and the complete assembled assistant response
errorA safe failure event if classification, session lookup, or generation fails
Technical nuance: Gemini controls internal token batching, so the API exposes incremental text chunks rather than guaranteed tokenizer boundaries.
04 · Memory

Conversation state survives requests

1. Load

Retrieve all earlier user and assistant messages for the session.

2. Generate

Send structured history plus the latest question to Gemini.

3. Stream

Forward the response incrementally without waiting for completion.

4. Persist

Save both sides of the exchange for the next turn.

Demo proof: ask a follow-up containing “it” and show that Gemini understands the earlier subject.
05 · User experience

The interface reads the room

Adaptive modes

Code, creative, study, and general themes respond to conversation intent.

Automatic organization

Sessions receive generated titles and stable majority-vote categories.

Visible streaming

Live chunks, first-chunk latency, completion time, and suggested follow-ups.

Classification is a separate lightweight Gemini call; failure falls back to the general mode and never blocks chat generation.
